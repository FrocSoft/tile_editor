# Tile Editor

아이패드 미니에 맞춘 **오프라인 동작** 8비트 글리치 그래픽 편집기.
이미지를 8×8 픽셀 타일로 분절하고, 분절된 타일(또는 타일 블록)을 브러시로
BG/스프라이트 두 레이어에 찍으며 글리치 아트를 만든다.

서버 없이 정적 파일만으로 동작하는 PWA — 모든 작업물은 기기(IndexedDB)에 저장된다.

## 아이패드에 설치하기 (오프라인 사용)

1. 리포 설정에서 **GitHub Pages**를 켠다: Settings → Pages → Source를 **GitHub Actions**로 선택.
   main 브랜치에 푸시하면 `.github/workflows/pages.yml`이 자동 배포한다.
2. 아이패드 사파리에서 배포된 주소(`https://<계정>.github.io/tile_editor/`)를 연다.
3. 공유 버튼 → **"홈 화면에 추가"**.
4. 이후에는 와이파이 없이도 홈 화면 아이콘으로 실행 가능 — Service Worker가
   앱과 에셋 전체를 기기에 캐싱해 둔다.

## 사용법

- **🗂 에셋**: `assets/` 폴더의 스프라이트를 폴더별로 탐색해 선택하면 8×8로 분절된다.
  사진/파일에서 직접 가져오기도 가능.
- **소스 패널**(하단): 탭 = 타일 하나 선택, **드래그 = 사각 범위 선택** → 타일 블록 브러시.
  "캔버스에 배치"로 소스 전체를 활성 레이어에 깔 수 있다.
- **레이어**: `BG`/`SPR` 세그먼트로 활성 레이어 전환, 👁 버튼으로 레이어별 표시 토글.
  스프라이트 레이어가 배경 위에 합성된다.
- **캔버스 크기**: 상단 `W±`/`H±` 버튼으로 1타일 단위 조절.
- **도구**: 🖌 스탬프 · ▦ 패턴 채우기(브러시 블록으로 드래그 영역 타일링) ·
  ⬚ 선택(드래그 이동/복제/삭제/브러시화) · 👆 타일 집기(탭한 칸의 타일을 브러시로) ·
  ◽️ 지우개 · ⇆ 행/열 밀기(글리치) · 🎲 랜덤 교란
- **브러시 변환**: 브러시 미리보기 옆 ⟳(90° 회전) ⇋(좌우 반전) ⇅(상하 반전)
- **🎨 NES 팔레트**: 실제 NES PPU 방식 — 마스터 팔레트(기본 NES 54색, `.pal`/`.hex`/`.gpl`
  파일로 교체 가능)에서 색을 골라 서브팔레트(3색+공용 배경색) 4개를 구성.
  "적용"은 선택 영역(없으면 전체)의 타일을 서브팔레트 4색으로 양자화(타일당 4색 제한 재현),
  "색 대치"는 캔버스를 탭해 집은 원본 색을 마스터 팔레트의 원하는 색으로 교체
- **제스처**: 한 손가락/애플펜슬 = 도구, 두 손가락 = 핀치 줌·팬
- **⬇︎**: PNG 내보내기 (4× 확대)

## 에셋 추가하기

```
assets/
  <폴더명>/
    sprite1.png
    sprite2.png
```

PNG를 넣은 뒤 인덱스를 갱신하고 커밋한다:

```sh
python3 tools/build_asset_index.py
```

(GitHub Actions 배포 시에도 자동으로 재생성된다.)

샘플 에셋과 앱 아이콘은 `python3 tools/generate_samples.py`로 다시 만들 수 있다.

## 로컬 실행

```sh
python3 -m http.server 8000
# http://localhost:8000
```

Service Worker는 HTTPS 또는 localhost에서만 동작한다.
